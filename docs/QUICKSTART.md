## üöÄ Installation (3 Steps)

### Step 1: Clone & Install

```bash
# Clone the repository
git clone <your-repo-url>
cd isa-protocol-v1

# Run automated setup
chmod +x scripts/setup.sh
./scripts/setup.sh
```

The setup script will:
- Install dependencies
- Generate security keys
- Create configuration files
- Set up both Core and Public CLI

### Step 2: Configure MongoDB

**Option A: Local MongoDB**
```bash
# Start MongoDB
sudo systemctl start mongodb
```

**Option B: MongoDB Atlas (Cloud)**
1. Go to [mongodb.com/cloud/atlas](https://www.mongodb.com/cloud/atlas)
2. Create free cluster
3. Get connection string
4. Update `core/.env`:
   ```env
   MONGO_URI=mongodb+srv://username:password@cluster.mongodb.net/isa-protocol
   ```

### Step 3: Start Services

```bash
# Terminal 1: Start Core Server
cd core
bun start

# Terminal 2: Start Public CLI
cd public-cli
bun start
```

**Or use PM2 for background processes:**
```bash
./scripts/start-all.sh
```

Via API:


## üîß Common Issues

### Bot Not Responding

**Check if services are running:**
```bash
pm2 status
```

**Check logs:**
```bash
pm2 logs isa-client
```

**Restart:**
```bash
pm2 restart all
```

### QR Code Not Showing

**Make sure you're in the right directory:**
```bash
cd public-cli
bun start
```

**Delete old session:**
```bash
rm -rf auth_info
bun start
```

### Database Connection Error

**Check MongoDB:**
```bash
sudo systemctl status mongodb
```

**Start MongoDB:**
```bash
sudo systemctl start mongodb
```

### API Key Mismatch

Make sure the `API_KEY` in both `.env` files is the same:
```bash
# Check Core API key
cat core/.env | grep API_KEY

# Check CLI API key
cat public-cli/.env | grep API_KEY
```
